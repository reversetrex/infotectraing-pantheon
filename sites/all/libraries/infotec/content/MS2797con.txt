<p><strong> Unit 1: Designing a Reporting Solution<br>
</strong>In this Unit, students will learn how to identify user requirements and business requirements for a reporting solution. They will then learn how to match the requirements of the user to Reporting Services technologies.</p>
<p> Units</p>
<ul>
  <li> Evaluating Business Requirements for a Reporting Solution </li>
  <li>Designing for Reporting Services </li>
</ul>
<p> Lab 1: Designing a Reporting Solution</p>
<ul>
  <li> Identifying Reporting Solution Users </li>
  <li>Creating a Reporting Solution Specification </li>
</ul>
<p> After completing this Unit, students will be able to:</p>
<ul>
  <li> Evaluate the business requirements for a reporting solution. </li>
  <li>Design a Reporting Services implementation. </li>
</ul>
<p><strong> Unit 2: Developing a Reporting Solution <br>
</strong>In this Unit, students will first identify how best to develop reports using the powerful Business Intelligence Development Studio Report Designer tool. This will include a review of best practice for report layout, and how this is affected by the various rendering formats available. Secondly, they will identify the considerations for effective development of a report model to enable more experienced users to create their own reports.</p>
<p> Units</p>
<ul>
  <li> Developing Reports </li>
  <li>Developing Report Models </li>
</ul>
<p> Lab 2: Developing a Reporting Solution</p>
<ul>
  <li> Creating Reports from a Report Specification </li>
  <li>Working With Report Models </li>
</ul>
<p> After completing this Unit, students will be able to:</p>
<ul>
  <li> Create a Reporting Services solution. </li>
  <li>Create a report model and use it to create a report with Report Builder. </li>
</ul>
<p><strong> Unit 3: Integrating Reporting Services with Applications<br>
</strong>In this Unit, students will review the options for integrating Reporting Services reports into a custom application. They will look at using direct URL access, the report viewer controls provided by Microsoft Visual Studio 2005, the Reporting Services SharePoint Web Parts, and the Reporting Services Web services. Lastly, they will discuss which approach, or combination of approaches, might suit different BI applications and their different kinds of users.</p>
<p> Units</p>
<ul>
  <li> Integrating with Custom Applications </li>
  <li>Controlling Reporting Services by Using Web Services </li>
</ul>
<p> Lab 3: Integrating Reporting Services with Applications</p>
<ul>
  <li> Integrating Reports into an ASP.NET Application. </li>
  <li>Integrating a Report into a SharePoint Services Solution </li>
  <li>Implementing a Reporting Services Web Service Client </li>
</ul>
<p> After completing this Unit, students will be able to:</p>
<ul>
  <li> Integrate Reporting Services with custom applications. </li>
  <li>Design Web service clients for Reporting Services. </li>
</ul>
