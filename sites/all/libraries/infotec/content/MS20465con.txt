<p><strong>Module 1: Designing a Database Server Infrastructure</strong> - This module explains how to design an appropriate database server infrastructure for a given business application scenario - including how to decide between on-premise, cloud-based, and hybrid database servers, hardware capacity planning, considerations for storage hardware, and strategies for consolidating database server hardware.</p>
<p>Lessons</p>
<ul>
  <li>Planning a Database Server Infrastructure</li>
  <li>Planning Server Hardware</li>
  <li>Considerations for Database Server Consolidation</li>
  <li>Managing Server Resources in a Consolidated Database Infrastructure</li>
</ul>
<p><strong>Module 2: Designing a Logical Database Schema</strong> - This module explains how to design a logical schema for a database based on application requirements. This includes planning the level of normalization, and schema and table design, and the use of views.</p>
<p>Lessons</p>
<ul>
  <li>Relational Database Design Techniques</li>
  <li>Planning Schemas and Tables</li>
</ul>
<p><strong>Module 3: Designing a Physical Database Implementation</strong> - This module explains how to design the physical implementation of a database for a given set of requirements. The design will include data files, log files, filegroups, and data partitioning, as well as whether or not to use data compression.</p>
<p>Lessons</p>
<ul>
  <li>Planning Files and Filegroups</li>
  <li>Planning a Partitioning Data</li>
  <li>Planning Compression</li>
</ul>
<p><strong>Module 4: Incorporating Data Files into Databases</strong> - <p>This modules discusses how to consider options for including data files in a database design.</p>
<p>Lessons</p>
<ul>
  <li>Considerations for Working with Data Files in SQL Server 2012</li>
  <li>Implementing FileStream and FileTables</li>
  <li>Searching Data Files</li>
</ul>
