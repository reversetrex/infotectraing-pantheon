<p><strong>Unit 1: Introduction to Designing SQL Server Security</strong></p>
<p>This Unit introduces the principles and methodology of designing SQL Server security. This Unit also explains the benefits 

of having a security policy in place and the process of creating a security policy. In addition, this Unit teaches you the 

importance of monitoring the security of SQL Server.<br>
  <br>
  Units</p>
<ul>
  <li>Principles of Database Security</li>
  <li>Methodology for Designing a SQL Server Security Policy</li>
  <li>Monitoring SQL Server Security<br>
    </li>
</ul>
<p>After completing this Unit, students will be able to:</p>
<ul>
  <li>Explain the principles of SQL Server security.</li>
  <li>Describe the methodology to design a SQL Server security policy.</li>
  <li>Explain the importance of monitoring SQL Server security.</li>
</ul>
<p><strong>Unit 2: Designing a SQL Server Systems Infrastructure Security Policy</strong></p>
<p>This Unit provides the guidelines for implementing server-level security using authentication methods. This Unit also 

provides the knowledge required to develop a Microsoft Windows server-level security policy. To enable you to do this, this 

Unit provides the guidelines to create password policy and determine service accounts permissions. In addition, this Unit 

explains how to select an appropriate encryption method to develop a secure communication policy. This Unit also explains the 

monitoring standards for SQL Server.</p>
<p>Units</p>
<ul>
  <li>Integrating with Enterprise Authentication Systems</li>
  <li>Developing Windows Server-Level Security Policies</li>
  <li>Developing a Secure Communication Policy</li>
  <li>Defining SQL Server Security Monitoring Standards<br>
    </li>
</ul>
<p>Lab 2A: Designing a SQL Server Systems Infrastructure Security Policy</p>
<ul>

  <li>Developing a Secure Communication Policy</li>
  <li>Integrating SQL Server Security Within the Active Directory Environment</li>
  <li>Integrating SQL Server Security With Firewall Configurations</li>
  <li>Discussing Systems Infrastructure Security Integration</li>
</ul>
<p>Lab 2B: Creating an Infrastructure Security Inventory</p>
<ul>

  <li>Auditing the Windows Local Password Policy</li>
  <li>Auditing SQL Server Service Accounts</li>
  <li>Monitoring Security at the Enterprise and Server Levels</li>
</ul>
<p>After completing this Unit, students will be able to:</p>
