<p><strong>Module 5: Creating Multidimensional Databases</strong></p>
<p>The fundamental purpose of using SQL Server Analysis Services online analytical processing (OLAP) solutions is to build cubes that you can use to perform complex queries and return the results in a reasonable time. This module provides an introduction to multidimensional databases and introduces the core components of an OLAP cube.</p>
<ul>
  <li>Introduction to Multidimensional Analysis</li>
  <li>Creating Data Sources and Data Source Views</li>
  <li>Creating a Cube</li>
  <li>Overview of Cube Security</li>
</ul>
<p><strong>Module 6: Working with Dimensions</strong></p>
<p>In SQL Server Analysis Services, dimensions are a fundamental component of cubes. This module provides an insight into the creation and configuration of dimensions and dimension hierarchies.</p>
<ul>
  <li>Configuring Dimensions</li>
  <li>Defining Attribute Hierarchies</li>
  <li>Sorting and Grouping Attributes</li>
</ul>
<p><strong>Module 7: Working with Measures and Measure Groups</strong></p>
<p>A measure represents a column that contains quantifiable data, usually numeric, that you can aggregate. This module describes measures and measure groups. The module also explains how you can use measures to define fact tables   and associate dimensions.</p>
<ul>
  <li>Working with Measures</li>
  <li>Working with Measure Groups</li>
</ul>
<p><strong>Module 8: Introduction to MDX</strong></p>
<p>Multidimensional Expressions (MDX) is the query language that you use to work with and retrieve multidimensional data in SQL Server Analysis Services. This module describes the fundamentals of MDX. It also explains how to build   calculations, such as calculated members and named sets.</p>
<ul>
  <li>MDX Fundamentals</li>
  <li>Adding Calculations to a Cube</li>
  <li>Using MDX to Query a Cube</li>
</ul>

<p><strong>Module 12: Implementing an Analysis Services Tabular Data Model</strong></p>
<p>With SQL Server 2012, you can install Analysis Services in Tabular mode and create tabular data models that information workers can access by using tools such as Excel and Power View. This module describes Analysis Services tabular data models and explains how to develop a tabular data model by using the SQL Server Data Tools.</p>
<ul>
  <li>Introduction to Analysis Services Tabular Data Model Projects</li>
  <li>Developing an Analysis Services Tabular Data Model in SQL Server Data Tools</li>
</ul>
<p><strong>Module 13: Creating Data Visualizations with Power View</strong></p>
<p>SQL Server 2012 introduces Power View, a SharePoint-based data exploration tool that provides a way for information workers to interactively create data visualizations that help them to better understand the data that they are working with. This module introduces Power View and describes how you can use it to create a range of different types of reports quickly and easily.</p>
<ul>
  <li>Introduction to Power View</li>
  <li>Visualizing Data with Power View</li>
</ul>
<p><strong>Module 14: Performing Predictive Analysis with Data Mining</strong></p>
<p>SQL Server Analysis Services includes data mining tools that you can use to identify patterns in your data, helping you to determine why particular things happen and to predict what will happen in the future. This module introduces data mining, describes how to create a data mining solution, how to validate data mining models, how to use the Data Mining Add-ins for Excel, and how to incorporate data mining results into Reporting Services reports.</p>
<ul>
  <li>Overview of Data Mining</li>
  <li>Creating a Data Mining Solution</li>
  <li>Validating a Data Mining Solution</li>
  <li>Consuming a Data Mining Solution</li>
</ul>
